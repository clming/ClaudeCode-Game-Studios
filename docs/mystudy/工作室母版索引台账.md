# 工作室母版索引台账

## 1. 这份文件的作用

这份文件不是运行时规则，也不是技能说明。

它的作用是：

- 作为《游戏工作室协作模板》的母版台账
- 记录当前模板最关键的数量、入口、索引文件和事实来源
- 帮助你以后定制《A 游戏工作室》《B 游戏工作室》时，先有一个稳定基准

你可以把它理解成：

> 这套 AI 工作室母模板的“资产总表”和“核对清单”。

---

以下数量应以仓库当前实际目录为准：

- Agents：49

## 4. 这份台账最重要的价值

如果你未来要基于这个母模板分化新工作室，真正最容易漂移的不是“理念”，而是这些基础事实：

- 新增了 Agent，但索引没补
- 新增了 Skill，但 README 没补
- 新增了 Rules，但 `rules-reference.md` 没补
- 改了目录结构，但学习文档还在引用旧路径
- 模板数量变了，但对外说明还停留在旧版本

所以这份台账的价值，不在于它写得多，而在于它能提醒你：

> 任何一次模板升级，都应该先核对“真实目录”和“索引入口”是否同步。

---

## 5. 母模板维护时的单一事实源原则

这是最该坚持的一条：

## 5.1 真实目录是第一事实源

优先级最高的是目录现实：

- `.claude/agents/`
- `.claude/skills/`
- `.claude/rules/`
- `.claude/docs/templates/`
- `.claude/hooks/`

如果文档和目录不一致，先相信目录，再修文档。

---

## 5.2 索引文件是第二事实源

索引文件负责让人和 Agent 更容易发现这些内容：

- `.claude/docs/agent-roster.md`
- `.claude/docs/skills-reference.md`
- `.claude/docs/rules-reference.md`
- `.claude/docs/hooks-reference.md`

这些文件应尽量和真实目录同步。

---

## 5.3 README / quick-start 是第三事实源

这两类文件是对外入口：

- `README.md`
- `.claude/docs/quick-start.md`

它们最重要的作用不是“最完整”，而是“入口正确”。

也就是说：

- 数字不能错
- 关键入口不能漏
- 新增的重要 Agent / Skill 不能长期处于“仓库存在但入口不可发现”的状态

---

## 6. 以后每次升级母模板，优先核对什么

建议固定按这个顺序核对：

1. 真实目录数量有没有变
2. 索引文件有没有同步
3. README 和 quick-start 有没有同步
4. 学习文档是否还在引用旧说法
5. 是否出现“规则文件”和“解释文件”重复制定同一规则

---

## 7. 对你最有用的维护动作

如果你以后维护这套母模板，最值得优先做的是：

1. 每次新增 Agent 后，更新：
   - `.claude/docs/agent-roster.md`
   - `.claude/docs/agent-coordination-map.md`
   - 必要时更新 `README.md` 和 `.claude/docs/quick-start.md`
2. 每次新增 Skill 后，更新：
   - `.claude/docs/skills-reference.md`
   - 必要时更新 `README.md` 和 `.claude/docs/quick-start.md`
3. 每次新增 Rule 后，更新：
   - `.claude/docs/rules-reference.md`
   - 必要时更新学习文档中的目录说明
4. 每次新增模板后，更新：
   - `README.md`
   - `.claude/docs/quick-start.md`
5. 每次修改会话与交接规则后，检查：
   - `.claude/docs/context-management.md`
   - `.claude/docs/session-state-rules.md`
   - `docs/COLLABORATIVE-DESIGN-PRINCIPLE.md`
   - `docs/mystudy/` 下的解释文档

## 8. 当你开始定制《A 游戏工作室》《B 游戏工作室》时，怎么用这份台账

最稳的做法不是一上来就改很多 Agent，而是：

1. 先把这份台账当作母版基线
2. 确定你的新项目需要新增什么角色、技能、规则和目录
3. 每新增一项，就检查它对应的索引入口有没有补
4. 最后再检查学习文档是否需要同步改写

这样你做出来的不是“改了很多文件的新项目”，而是：

> 一个结构仍然清楚、入口仍然准确、后续还能继续维护的专用 AI 工作室。
